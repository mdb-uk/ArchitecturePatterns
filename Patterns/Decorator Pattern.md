# Content

This is a structural design pattern that lets you attach new behaviours to objects by placing them inside special wrapper objects which contain the desired new behaviours.

This is an alternative to subclassing, which helps avoid exponential subclass bloat of having to subclass a large number of possible combinations - with the Decorator pattern you can simply wrap as many decorator classes as you need.
