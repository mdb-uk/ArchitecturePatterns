# Component based architecture

This pattern emphasises reuse or components and the separation of concerns. It involves building a system into smaller, manageable parts called Components. Each component encapsulates specific functionality and can be developed, tested and maintained independently. 

Benefits:

- Reduced development time
- Enhanced reliability
- Flexibility
- Replicability of components 

Challenges

- Large component based applications can get very complicated, and converting an existing large app to component based architecture can be challenging
- Maintenance of the potentially large number of components can be difficult

Examples

- Microservices - each service is a component that performs a specific function and communicates with other components via APIs
- UI Components - Frameworks like React and Angular use components to build front end applications
