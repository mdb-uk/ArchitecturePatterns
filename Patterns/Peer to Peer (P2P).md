# Content

Decentralised computing modal where each node, or "peer", acts as both a client and a server. Every node in the network can both request and provide services, enabling direct resource sharing without relying on a central server.

### Key characteristics

- Decentralised
- Scalable
- Fault tolerant
- Resource sharing
- Autonomous

### Architectures
P2P networks can either be "Pure", which are completely decentralised, whereas "Hybrid" P2P networks have a combination of both centralised and decentralised architectures, often using central servers or "super peers" for coordination.

### Challenges

Security - because of the web of communications ensuring secure communication and data integrity can be challenging

Management - especially in a Pure P2P network it can be a challenge coordinating and managing resources
